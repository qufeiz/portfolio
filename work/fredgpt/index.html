01

The problem

The Federal Reserve publishes some of the most authoritative economic data and history in the world, but it is scattered across many disconnected systems: live time series in FRED, scanned historical documents in FRASER, working papers in Fed in Print, and policy history on FederalReserveHistory.org.

To answer a single real question — "What were the latest FOMC policy changes, and why?" — a user has to know which system holds the data, search each one separately, then manually stitch the numbers together with the surrounding policy context. The information is public, but it is not usable.

02

What it does

FredGPT is an agent-based GenAI platform that unifies and reasons over these diverse Federal Reserve sources behind a single chat interface. A user asks in natural language and gets back an accurate, explainable answer that combines:

  • Structured time series — the actual numbers pulled live from FRED (rates, unemployment by demographic, deficits, GDP, yield-curve indicators).
  • Unstructured history — relevant policy text and FOMC documents retrieved from FRASER and the Fed corpus.
  • Charts, citations, and metadata — every answer traceable back to its authoritative source, with the underlying series IDs and document provenance shown on demand.

It lowers the bar to economic literacy and raises transparency: answers are grounded in, and traceable to, official Federal Reserve sources. It serves researchers, educators, students, and the general public.

03

How it works

FredGPT is a modular multi-agent system built on retrieval-augmented generation. The core idea: a hybrid retriever that gets both keyword precision and semantic recall, fused into one ranked result the agent can reason over.

01 Frontend React chat experience served via AWS CloudFront.
02 Agent backend LangGraph / LangChain orchestrates conversation state and tool calls, iterating until it can assemble a grounded answer.
03 Tooling An MCP server on AWS Lambda exposes the data tools to the agent over a standardized, transport-agnostic interface.
04 Retrieval (core) BM25 keyword search in parallel with semantic vector search (Bedrock Titan Embed Text v2, 1024-d) over Amazon OpenSearch, fused with Reciprocal Rank Fusion. ~3,500 LOC TypeScript/Express, Dockerized, with API-key auth, rate limiting, health checks.
05 Safety AWS Bedrock guardrails block harmful or ungrounded responses while still allowing factual economic Q&A.

Guardrail testing and retrieval benchmarking were treated as first-class concerns, not afterthoughts.

04

Outcome

FredGPT demonstrates an end-to-end agentic pipeline that turns fragmented, expert-only Federal Reserve data into conversational, charted, source-cited answers. It improves economic literacy by explaining data in context, increases transparency by making every answer traceable to an authoritative source, and shows a credible, cloud-native deployment path on AWS for serving that capability to researchers, educators, students, and the public.

05
